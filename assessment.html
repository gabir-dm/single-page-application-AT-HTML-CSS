ASSESSMENT

Respostas para o Questionário

1) Quando é recomendando a criação de um site separado para mobile? Seria este o caso do site integrado de Interfaces - SPA?

Geralmente não é recomendado a criação de um site separado apenas para dispositivos mobile. É preferível a manipulação dos elementos através da folha de estilos CSS para a criação de regras específicas que atendam à responsividade adequada para cada tipo de dispositivo que possa ir a acessar o site - de preferência através de media queries em uma única folha de estilo, e não a criação múltiplas folhas de estilo separadas.
Sim, para o modelo de site SPA de interfaces, esse seria o jeito ideal de lidar com a responsividade para aparelhos mobile.

2) No caso de um CSS específico para print, como ele deve ser inserido?

Apesar de ser possível a criação de um arquivo CSS específico para impressões, a forma mais correta seria através da inserção de media queries (neste caso, "@media print") ao final da folha de estilos principal, tal qual é feito para as estilizações específicas de responsividade de tela.

3) O elemento audio permite como elemento‐filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quando devem ser utilizados?

O elemento-filho source, quando associado ao elemento audio, permite os atributos src (para definir o endereço de origem da mídia requisitada) e type (para definir o tipo de mídia vinculada e checar a possibilidade de sua execução).
Outros atributos presentes no elemento audio são autoplay (para que o audio comece a tocar assim que o elemento carregar), controls (para gerar botões de controle para o usuário), crossorigin (relacionado ao vínculo do endereço da mídia com o documento), disableremoteplayback (para impedir a reprodução do audio em dispositivos conectados remotamente ao dispositivo principal), loop (para o audio voltar ao começo e tocar novamente ao terminar sua reprodução), mute (gerando um valor booleano que controla se o áudio se inicia no mudo) e preload (que pode ser definido para que o navegador pre-carregue ou não a mídia para o cache).

4) O elemento video permite como elemento‐filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento video e quando devem ser utilizados?

O elemento-filho source, quando associado ao elemento video, permite os atributos src (para definir o endereço de origem da mídia requisitada) e type (para definir o tipo de mídia vinculada e checar a possibilidade de sua execução).
Outros atributos presentes no elemento video são autoplay (para que o vídeo comece a tocar assim que o elemento carregar), autopictureinpicture (valor booleano que controla se o elemento vídeo gera uma visualização pequena do vídeo no caso do usuário trocar de tela durante a reprodução do mesmo), controls (para gerar botões de controle para o usuário) e controlslist (que permite trocar quais botões de controle disponíveis para o usuário), crossorigin (relacionado ao vínculo do endereço da mídia com o documento), disableremoteplayback (para impedir a reprodução do vídeo em dispositivos conectados remotamente ao dispositivo principal), height e width (para definir a altura e largura da área de visualização do vídeo), loop (para o vídeo voltar ao começo e passar novamente ao terminar sua reprodução), mute (gerando um valor booleano que controla se o áudio do vídeo se inicia silenciado), preload (que pode ser definido para que o navegador pre-carregue ou não a mídia para o cache), poster (que pode linkar para a url de uma imagem estática a ser exibida enquanto o vídeo carrega) e playsinline (para garantir que o vídeo será reproduzido dentro de sua área de playback definida).

5) No caso do canvas, HTML 5 nunca, em momento algum, fez qualquer animação. Isso porquê o HTML 5 é uma simples linguagem de marcação. Dessa forma, o que é necessário para criar animações com HTML5? O que são APIs do HTML5?

É possível criar animações para o HTML através de certos atributos no CSS e utilizando Javascript para a aplicação de APIs.
APIs (Application Programming Interface) são bibliotecas de parâmetros e protocolos pre-estabelecidos que podem ser chamadas e utilizadas para auxiliar no desenvolvimento nas mais diferentes tarefas através de solicitações remotas feitas do computador do usuário ao servidor onde a API se encontra. É uma forma segura de implementar novos componentes à uma aplicação.

6) Como o CSS3 pode ser aplicado na aparência da multimídia HTML5. Demonstre com um exemplo.

O CSS pode ser aplicado através do vínculo da folha de estilos com o documento HTML em questão. Ao assinalar classes ou ids aos elementos multimídia a serem alterados, é possível utilizar os parâmetros de estilo do CSS para alterar diversos aspectos dos elementos referenciados, tal qual tamanho do elemento, seu posicionamento na página, seu comportamento em relação ao hover do mouse, a estilização e formatação de seu texto, e até mesmo algumas animações simples como zoom, flip, fade e etc.
